# StartPage — Стартовая страница в Docker с базой данных SQLite

Удобная стартовая страница для браузера с группировкой ссылок по блокам и поддержкой drag-and-drop перетаскивания. Данные сохраняются на сервере в базе данных **SQLite**.

---

## Особенности новой архитектуры с SQLite

1. **Серверное хранение**: Все блоки и ссылки хранятся на сервере в базе данных SQLite (`data/startpage.db`).
2. **Мгновенный интерфейс (Optimistic UI)**: При изменении порядка (Drag-and-Drop) или добавлении ссылок интерфейс обновляется мгновенно, а синхронизация с базой данных SQLite происходит асинхронно в фоновом режиме.
3. **Постоянные данные (Volumes)**: При запуске в Docker папка с базой данных монтируется на хост-компьютер, что гарантирует сохранность данных при обновлении или перезапуске контейнеров.

---

## Быстрый запуск с помощью Docker

### Требования
* Установленный [Docker](https://www.docker.com/) и `docker-compose`.

---

### 1. Режим разработки (Development)

В этом режиме запускается Next.js сервер в режиме разработки с поддержкой **Hot Reload**. База данных в режиме разработки создается внутри папки `app/data/`.

* **Команда для запуска**:
  ```bash
  docker compose up --build dev
  ```

* **Адрес в браузере**: `http://localhost:3000`
* **Особенности**:
  * Папка `app` примонтирована как Volume, поэтому изменения кода на хосте синхронизируются с контейнером.
  * Включен `WATCHPACK_POLLING=true` для стабильной авто-перезагрузки на Windows/WSL2.

---

### 2. Продакшен режим (Production)

В продакшен режиме приложение собирается в оптимизированный билд Next.js и запускается как полноценный Node.js сервер.

* **Команда для запуска**:
  ```bash
  docker compose up --build prod
  ```

* **Адрес в браузере**: `http://localhost:8080`
* **Особенности**:
  * Файл базы данных хранится в папке `./data/startpage.db` в корне проекта на вашем компьютере.
  * Настроен Docker Volume для папки `/app/data` внутри контейнера.

---

### Управление контейнерами (Полезные команды)

* **Запуск в фоновом режиме**:
  ```bash
  docker compose up -d <service_name>
  ```
  *(где `<service_name>` — это `dev` или `prod`)*

* **Остановка контейнеров**:
  ```bash
  docker compose down
  ```

* **Просмотр логов**:
  ```bash
  docker compose logs -f
  ```

## Развертывание без Docker (Локально)

Если вы хотите запустить проект локально в вашей системе:

1. Перейдите в папку с приложением:
   ```bash
   cd app
   ```
2. Установите зависимости:
   ```bash
   npm install
   ```
3. Запустите в режиме разработки:
   ```bash
   npm run dev
   ```
   *(база данных создастся по адресу `app/data/startpage.db`)*

4. Для запуска продакшен сервера локально:
   ```bash
   npm run build
   ```
   ```bash
   npm run start
   ```
   *(сервер запустится на порту `3000`)*
